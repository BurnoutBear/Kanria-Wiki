# 📖 Game Wiki

Public wiki repository. Single source of truth for both:
- 🌐 **Web** — served via GitHub Pages at `https://YOUR_USERNAME.github.io/YOUR_WIKI_REPO/`
- 🎮 **In-game** — embedded in the main Godot project via git submodule

---

## Repository structure

```
wiki/
├── index.html              # GitHub Pages landing page
├── 404.html                # Custom 404 page
├── coi-serviceworker.js    # Cross-origin isolation fix for GitHub Pages
├── assets/
│   ├── main.js             # All page JS (separate file — required by CSP)
│   └── style.css           # Wrapper page styles
├── wiki_export/            # ← Godot HTML5 export goes here
│   ├── .gitkeep            # Placeholder — delete once you export
│   ├── index.html          # (generated by Godot)
│   ├── index.js            # (generated by Godot)
│   ├── index.wasm          # (generated by Godot)
│   ├── index.pck           # (generated by Godot)
│   └── index.audio.worklet.js  # (generated by Godot)
├── .gitignore
└── README.md
```

## First-time setup

### 1. Create the GitHub repo

Create a **public** repository on GitHub. Push these files to the `main` branch.

### 2. Enable GitHub Pages

1. Go to **Settings → Pages**
2. Source: **Deploy from a branch**
3. Branch: `main`, folder: `/` (root)
4. Save

Your site will be live at `https://YOUR_USERNAME.github.io/YOUR_WIKI_REPO/` within ~60 seconds.

### 3. Add as a submodule in your main game

Inside the root of your private Godot project:

```bash
git submodule add https://github.com/YOUR_USERNAME/YOUR_WIKI_REPO wiki
git commit -m "add wiki submodule"
```

The wiki files are now available in Godot at `res://wiki/wiki_export/`.

## Updating the wiki

1. Make changes in the **Godot wiki project** (your private repo)
2. In the Godot export dialog, set the export path to `path/to/wiki/wiki_export/`
3. Export → HTML5 (overwrite existing files)
4. Commit and push **this repo**:
   ```bash
   cd wiki
   git add wiki_export/
   git commit -m "update wiki export"
   git push
   ```
5. In the main game repo, update the submodule pointer:
   ```bash
   cd ..         # back to main game root
   git add wiki
   git commit -m "update wiki submodule"
   git push
   ```

## Customisation

- **Game name**: Update the `<title>` and `<h1>` in `index.html` and `404.html`
- **Source link**: Replace `YOUR_USERNAME/YOUR_WIKI_REPO` in `index.html`'s footer
- **Colours/fonts**: Edit CSS variables at the top of `assets/style.css`

## Security notes

| Measure | Where | Why |
|---|---|---|
| Content Security Policy | `<meta>` in `index.html` / `404.html` | Prevents XSS on the wrapper page. `script-src 'self'` means only files from this repo can run. |
| No inline JS | `assets/main.js` | Required by `script-src 'self'`. All logic in an external file. |
| `rel="noopener noreferrer"` | External links | Prevents opened tabs accessing `window.opener` (tab-napping). |
| `referrerpolicy="no-referrer"` | iframe | The wiki export doesn't need to know the parent URL. |
| `allow="fullscreen; pointer-lock"` | iframe | Grants only the Permissions Policy features Godot actually needs. |
| No `sandbox` on iframe | intentional | Godot requires same-origin access for IndexedDB (save data) and SharedArrayBuffer. Sandbox without `allow-same-origin` would break these. |
| coi-serviceworker reload guard | `coi-serviceworker.js` | Prevents the infinite reload loop that occurs if the SW activates but `navigator.serviceWorker.controller` is still null after reload. |
| `textContent` only in error messages | `assets/main.js` | Error messages are never inserted via `innerHTML`, preventing DOM injection. |

### If you move to Netlify or Vercel

You can replace the `coi-serviceworker.js` approach with proper HTTP headers,
which is more reliable:

**`netlify.toml`**
```toml
[[headers]]
  for = "/*"
  [headers.values]
    Cross-Origin-Opener-Policy   = "same-origin"
    Cross-Origin-Embedder-Policy = "require-corp"
```

**`vercel.json`**
```json
{
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        { "key": "Cross-Origin-Opener-Policy",   "value": "same-origin" },
        { "key": "Cross-Origin-Embedder-Policy",  "value": "require-corp" }
      ]
    }
  ]
}
```

If using these, you can remove the `<script src="coi-serviceworker.js">` tag from `index.html`.
